Overview

The Jenkins Remoting project provides a remote communication mechanism that allows Jenkins agents to communicate with the master. It is an integral part of Jenkins' distributed build system, enabling data exchange and command execution over a network.

Jenkins Remoting is designed to be lightweight and provide fast communication to enable efficient operation in Jenkins clusters.

Key Features

  • Cross-platform support (Windows, macOS, Linux)
  • Secure communication using encrypted channels
  • Efficient serialization and deserialization of objects
  • Lightweight architecture for low resource usage
  • Automatic reconnection and retry in case of failure

Installation

To install Jenkins Remoting, you can use the following steps:

# Download and install Jenkins Remoting JAR
wget https://repo.jenkins-ci.org/releases/org/jenkins-ci/main/remoting/4.15/remoting-4.15.jar

# Run the agent with the master
java -jar remoting-4.15.jar -jnlpUrl http:///computer//slave-agent.jnlp

Make sure your Jenkins master is running and you have the appropriate permissions to add agents to the system.

Usage

Once Jenkins Remoting is installed, it automatically handles communication between Jenkins agents and the master. Agents can be used to run build jobs remotely, improving scalability and efficiency in large Jenkins environments.
